Setting up development environment

First time setup

If you don't already have node.js installed, install it on your system.

The development environment is managed with a CLI, buster-dev-tools. Install it using the node.js package manager.

npm install -g buster-dev-tools

The tool will install all the buster packages to the folder you choose.

mkdir buster
cd buster
buster-dev-tools install

You also need to set some environment variables. {path to buster} is the path to the folder where you ran buster-dev-tools install.

NODE_PATH={path to buster}
PATH={path to buster}/buster/bin:$PATH

Refreshing all repositories

After some time you might want to update all the repos. To do so, use the update command.

cd buster
buster-dev-tools update

You need to cd to the directory where the packages are installed in order to use buster-dev-tools.

Contribution guidelines

Note: this section is work-in-progress

Post questions to the busterjs-dev mailing list

The busterjs-dev mailing list is a place for general discussion regarding development of Buster.JS. In many cases you can just post an issue to the issue tracker and the discussion can happen there. But you can also discuss on the mailing list if you're more comfortable with that. For example, you might prefer to use e-mail for discussion, or you want to discuss your contribution before posting it to the issue tracker.

Always create an issue for your contribution

We probably won't notice your contribution if all you do is submit a pull request on Github. But we monitor the issue tracker, so you should always post an issue when you create a pull request on GitHub, a merge request on Gitorious, or e-mail us a patch. Just link to your pull request(s) and explain your contribution.

Not creating pull requests are also fine. Create an issue and link to the feature branches and/or commits you want to contribute, be it on GitHub, Gitorious, or a link to a tarball of patches.
